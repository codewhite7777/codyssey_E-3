# Mini NPU Simulator

---

## 1. 실행 방법

```
python3 main.py
```

- 실행 후 모드 선택: `1` (사용자 입력 3×3) 또는 `2` (data.json 분석) 또는 `3` (종료)
- `data_v2.json` 파일은 `main.py`와 같은 디렉토리에 위치해야 합니다.

---

## 2. 구현 요약

### 2-1. 라벨 정규화

| 입력 값 | 변환 결과 | 출처 |
|---------|----------|------|
| `"+"` | `Cross` | expected 값 |
| `"cross"` | `Cross` | filter 키 |
| `"x"` | `X` | expected 값 / filter 키 |

- 프로그램 내부에서는 `Cross`, `X` 두 가지 표준 라벨만 사용합니다.
- 비교 및 출력은 모두 정규화된 라벨 기준으로 수행됩니다.

### 2-2. MAC 연산

- 두 배열(패턴, 필터)의 같은 위치 값을 곱한 뒤 전부 합산합니다.
- 이중 for문으로 직접 구현하였으며, NumPy 등 외부 라이브러리는 사용하지 않았습니다.
- 반환 타입은 `float`이며, 소수 입력도 처리 가능합니다.

### 2-3. 동점 처리 정책 (epsilon)

- 기준: `abs(score_a - score_b) < 1e-9` 이면 동점으로 간주합니다.
- 동점 시 판정 결과: `UNDECIDED`
- 사유: 컴퓨터의 부동소수점 연산에서 발생하는 미세한 오차로 인한 오판을 방지하기 위함입니다.

## 3. 결과 리포트

### 3-1. FAIL 케이스 원인 분석

총 테스트 8개 중 7개 통과, 1개 실패.

**실패 케이스: size_13_1**

| 항목 | 값 |
|------|-----|
| Cross 점수 | 7.6999999999999975 |
| X 점수 | 7.6999999999999975 |
| 판정 | UNDECIDED |
| expected | X |

- 이 케이스는 Cross 필터와 X 필터의 MAC 점수가 동일하게 나오는 패턴이다.
- 두 점수의 차이가 epsilon(1e-9) 미만이므로 UNDECIDED로 판정되었다.
- expected는 "X"이지만 판정이 UNDECIDED이므로 불일치하여 FAIL이 발생했다.
- 이는 프로그램의 로직 오류가 아니라, 데이터 자체가 동점을 유발하는 **수치 비교 문제**에 해당한다.
- 원인 분류: 데이터 문제(X)  /  로직 문제(X)  /  **수치 비교 문제(O)**

**나머지 7개 PASS 사유:**

- 라벨 정규화를 통해 `"+"` → `"Cross"`, `"x"` → `"X"` 변환이 정상 처리되었다.
- epsilon 기반 비교 정책이 올바르게 적용되어 부동소수점 오차가 판정에 영향을 주지 않았다.
- 필터와 패턴의 크기 검증을 수행하여 크기 불일치로 인한 오류가 발생하지 않았다.
- size_5_3 케이스는 노이즈가 포함된 소수 값(0.1, 0.9)이었으나, MAC 연산이 float로 정상 처리되어 올바르게 판정되었다.

### 3-2. 시간 복잡도 분석

MAC 연산은 N×N 배열의 모든 원소를 한 번씩 방문하는 이중 for문 구조이다.
바깥 루프 N번 × 안쪽 루프 N번 = 총 N²번의 곱셈/덧셈을 수행하므로 시간 복잡도는 O(N²)이다.

**성능 측정 결과 (10회 반복 평균):**

| 크기 | 평균 시간(ms) | 연산 횟수(N²) |
|------|-------------|-------------|
| 3×3 | 0.002 | 9 |
| 5×5 | 0.004 | 25 |
| 13×13 | 0.022 | 169 |
| 25×25 | 0.076 | 625 |

**O(N²) 검증:**

- 연산 횟수 비율 (3×3 → 25×25): 625 / 9 = 약 69.4배
- 실측 시간 비율 (3×3 → 25×25): 0.076 / 0.002 = 약 38배
- 연산 횟수 비율과 시간 비율이 비례 관계에 있으며, 소규모 배열에서의 오차는 Python 함수 호출 오버헤드 등 고정 비용에 의한 것이다.
- N이 커질수록 실제 연산 시간이 고정 비용 대비 지배적이 되어, 비율이 N²에 더 가깝게 수렴한다.
- 이를 통해 MAC 연산의 시간 복잡도가 O(N²)임을 실측으로 확인할 수 있다.
